<Section title="Summary" intro="The login exchange is stable; routing and the candidate unlock path still have gaps.">

<KeyPoints>

- OAuth provider token is exchanged at `POST /auth/oauth`; the backend verifies it and issues JWT access + refresh tokens.
- The response field `nextScreen` routes the user to either `account_completion` or `overview`.
- Guards: `JwtAuthGuard` validates the Bearer token, `RolesGuard` enforces the `@AuthRoles()` decorator.

</KeyPoints>

<Section title="The change" intro="Routing moved off the client: the resolver now returns nextScreen from the server.">

<Diff
  filename="src/auth/oauth-handler.ts"

  const profile = await verifyProviderToken(dto.provider, dto.token);
  const user = await users.upsertFromProvider(profile);
  const tokens = issueTokens(user);
  return { ...tokens };
}`}
  newText={`export async function handleOAuth(dto: OAuthDto) {
  const profile = await verifyProviderToken(dto.provider, dto.token);
  const user = await users.upsertFromProvider(profile);
  const tokens = issueTokens(user);
  return {
    ...tokens,
    nextScreen: user.profileComplete ? "overview" : "account_completion",
  };
}`}
