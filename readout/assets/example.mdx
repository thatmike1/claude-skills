<Section title="Summary" intro="The login exchange is stable; routing and the candidate unlock path still have gaps.">

<KeyPoints>

- OAuth provider token is exchanged at `POST /auth/oauth`; the backend verifies it and issues JWT access + refresh tokens.
- The response field `nextScreen` routes the user to either `account_completion` or `overview`.
- Guards: `JwtAuthGuard` validates the Bearer token, `RolesGuard` enforces the `@AuthRoles()` decorator.

</KeyPoints>

<Section title="Flow" intro="From provider token to landing screen, the backend decides everything in one pass.">

<Diagram code={`flowchart TD
  A[OAuth provider token] --> B[POST /auth/oauth]
  B --> C{User exists?}
  C -->|no| D[Create user]
  C -->|yes| E[Issue JWT access + refresh]
  D --> E
  E --> F{nextScreen}
  F -->|account_completion| G[Onboarding: role + profile]
